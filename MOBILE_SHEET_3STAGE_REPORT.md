# 手机端 2D 校园导览 · 三段式交互改进报告

**日期**：2026-07-08

---

## 改动概述

手机端 `/campus/2d` 底部面板从原来的两态（收起/展开）改为三态（收起/展开/详情），解决两个问题：
1. 上拉高度太矮（34%），列表看不全
2. 点地点弹出详情卡片遮住整个地图

## 交互流程

```
[收起态] 底部 44px 横条 → 「地点列表 · 30 个结果」
    │
    │ 上滑 > 16px
    ▼
[展开态] 占屏 70%
    ├── 搜索框
    ├── 分类标签
    └── 可滚动完整地点列表
    │
    │ 点击「图书馆」
    ▼
[详情态] 占屏 30%
    ├── 手柄显示「图书馆」
    ├── 图标 + 名称 + 分类 + 描述（最多 3 行）
    ├── [去这里] [返回列表] 两个按钮
    └── 地图定位到图书馆，marker 高亮
    │
    │ 上滑 / 点「返回列表」
    ▼
[展开态] 回到完整列表，选中地点保持高亮
```

## 边界情况

| 场景 | 预期行为 |
|------|----------|
| 初始加载 | 收起态，无详情卡片 |
| 展开列表 → 切换分类 | 筛选结果更新，选中地点如在结果中则保持高亮 |
| 详情态 → 上滑 | 回到展开态完整列表 |
| 详情态 → 点「去这里」 | 触发路线规划，sidebar 保持在详情态，地图绘制路线 |
| 详情态 → 下拉 | 保持在详情态（不收起） |
| 搜索框下拉选点 | 走 POI 搜索路径（`pickDestination`），不受 sidebar 改动影响 |
| 桌面端（>1100px） | sidebar 固定分栏布局，三态 CSS 仅在 ≤720px 生效，桌面不变 |

## 不变部分

- **桌面端 sidebar + detail 面板**：判断条件 `> 1100px`，完全不受影响
- **POI 搜索下拉**：走 `pickDestination` 独立路径
- **校准模式**：sidebar 不可见
- **GPS / 宿舍定位按钮**：定位逻辑不减
- **触摸手势**：阈值 16px、跟手反馈、60px 上限不变
