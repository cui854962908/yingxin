## 一、系统概览

迎新系统是一个前后端分离的全栈 Web 应用。用户规模为每年几千新生，使用周期集中在迎新季（7 月至 9 月），小团队开发维护，按年迭代。

### 1.1 产品定位（必读）

**一句话：** 2026 级新生迎新门户 — **认识牧院 · 指导报到 · 智能答疑**。

**我们是什么**

| 能力 | 做什么 | 主要模块 |
|------|--------|----------|
| **认识牧院** | 向新生与家长介绍学校、学院与社团 | 牧院大百科、学院概况、社团招新 |
| **报到指导** | 发布报到流程、新生攻略与校园信息 | 报到须知、新生攻略、校园导览 |
| **答疑互助** | 减少重复咨询，支持 AI 与同学互助 | FAQ、小信 AI、问牧墙 |

**面向谁**

- **游客**：无需登录，可浏览「认识牧院」（默认牧院大百科）
- **新生（已登录）**：在上述能力基础上，查看个人报到信息（辅导员、宿舍等），使用问牧墙提问与互动
- **管理员 / 社团管理员**：维护公告、FAQ、学院介绍、社团等内容（不在 Web 内导入或管理学籍名册）

**我们不是什么**

- **不是**学籍管理、名册维护、批量导入学生的后台系统
- 学生登录名单通过 **运维脚本** 写入数据库；Web 端只做 **身份校验 + 信息展示 + 内容宣传与答疑**

**前端文案源：** `frontend/src/constants/product.ts`（登录页、侧栏访客标签等与本文档对齐）

### 1.2 技术概览

**产品边界（技术侧）：** Web 端负责内容展示与问答；学生数据只读展示，不提供 Web 内名册 CRUD。

### 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | Vue 3 (Composition API + TypeScript) + Vite |
| 后端 | FastAPI (Python 3.12+) + SQLAlchemy + Alembic |
| 数据库 | PostgreSQL 16 + pgvector 向量扩展 |
| AI 对话 | DeepSeek API (deepseek-v4-flash) |
| AI 嵌入 | Ollama 本地部署 (bge-m3) |
| 语音合成 | Edge-TTS (zh-CN-XiaoyiNeural) |
| 工程化 | uv + Docker + Docker Compose |

## 二、登录与认证

### 2.1 登录页面

登录页设计为左右分栏——左侧深酒红品牌面板展示校徽、校训"尚严崇实·善知敏行"、2026 迎新标识；右侧为三字段表单：姓名、学号、身份证号（选填）。底部提供**游客浏览**入口，无需登录即可进入「认识牧院」模块（默认打开牧院大百科）。

### 2.2 登录流程

学生打开系统时，如果之前登录过（浏览器存有 token），自动调用 `/api/auth/me` 校验 token 有效性。token 有效则直接进入主界面；token 过期或首次访问则播放 2.8 秒欢迎动画后进入登录页。

提交后后端处理路径：

1. 查该学号是否已被锁定（5 次错误锁定 5 分钟，内存计数）
2. 按姓名 + 学号查 `students` 表：
   - 找不到记录 → 匹配失败
   - 找到记录，且 `id_number_hash` 有值 → 比对身份证号哈希
   - 找到记录，但 `id_number_hash` 为空 → 跳过身份证校验（支持未录入身份证号的学生）
3. 匹配成功 → 返回 JWT token + 完整个人信息，清除失败计数
4. 匹配失败 → 记录一次失败，返回剩余次数；达 5 次后锁定

### 2.3 认证安全机制

- JWT 密钥强度启动校验（`settings_guard.py`）：检查 `JWT_SECRET_KEY` 长度 ≥ 32 且非占位符；生产环境可设 `REQUIRE_SECURE_SETTINGS=true` 拒绝启动
- 身份证号加盐 SHA-256 哈希存储（`ID_NUMBER_SALT`），不可逆
- 登录接口限流：单 IP 每 60 秒最多 30 次（`rate_limit.py`）
- HTTP 安全头：X-Content-Type-Options、X-Frame-Options、Referrer-Policy（`http_middleware.py`）
- 上传文件大小限制 5MB（`http_middleware.py`）

### 2.4 登录后返回的信息

返回的个人信息包含：数据库主键 id、姓名、学号、班级、宿舍、辅导员（嵌套 name + phone）、班主任、代班列表、角色。前端将其存入 localStorage 并注入 Vue 的 provide/inject 体系。

## 三、主界面

### 3.1 区域说明

**侧边栏**（左侧 260px 固定宽度，移动端抽屉式滑出）：顶部校徽 + 校名中英文；用户区（首字头像 + 姓名 + 角色标签；访客显示「访客 · 信工门户」）；导航按钮（首页 / 校园公告 / 问题答疑 / 问牧墙 / **认识牧院**）；底部退出登录（访客为「退出浏览」）。

**个人信息卡片**（首页上半部分，进入社团/问牧墙全屏模块后自动收起；**访客与认识牧院页不展示**）：金色环形头像（取姓名首字 + 酒红渐变背景）+ 2026 徽章；姓名 + 班级；学号 + 宿舍；辅导员（姓名 + 电话）、班主任、代班（显示所属班级）。

**内容区**（首页下半部分 / 社团页面全屏）：白色圆角卡片承载当前路由组件，带阴影和模块切换动画。

**小信悬浮助手**（全局）：右下角 Lottie 动画角色，可在屏幕内自由拖动，点击弹出深色科技风聊天面板。面板包含打字机效果逐字输出、语音播报开关（Edge-TTS 晓依音色）、快捷提问标签。

### 3.2 首页时间轴

首页时间轴以可视化进度条展示迎新关键节点（按时间顺序排列）：

1. 录取结果（7 月 20 日）
2. 学费缴纳（8 月 20 日）
3. 宿舍选择（8 月 28 日）
4. 入学体检（9 月 1 日）
5. 军训服领取（9 月 4 日）
6. 开学典礼（9 月 6 日）
7. 军训开始（9 月 8 日）
8. 军训汇演（9 月 20 日）
9. 正式上课（9 月 22 日）

节点以不同颜色区分：已完成（绿色）、进行中（酒红色脉冲动画）、待进行（米色）。时间轴可左右滑动/拖拽浏览，顶部显示"距离报到 X 天"倒计时卡。

### 3.3 首页服务入口

多个常用功能卡片：报到须知、认识牧院、宿舍选择、新生缴费、选课系统、校园导览、学校官网、新生攻略、AI 迎新助手。部分卡片（缴费、选课、官网）点击跳转外部学校业务平台。

## 四、功能模块

### 4.1 校园公告

所有用户均可浏览。管理员可在后台新增（标题 + 正文 + 分类 + 日期）和删除。公告分为三类：校园公告（campus）、报到须知（guide）、新生攻略（tips），后两类在独立的报到须知页面展示。公告数据同步写入 `documents` 向量表，供小信知识库检索。

### 4.2 问题答疑（FAQ）

所有用户均可浏览，展示问题、答案、关键词、分类。支持关键词搜索和分页浏览。管理员可增删改 FAQ、拖拽调整排序。FAQ 表有 `keywords` 和 `category` 字段，keywords 直接影响小信 FAQ 匹配命中效果。新增页支持同时填写关键词和分类。查询结果有进程内缓存，增删后自动失效。FAQ 同样自动生成向量嵌入供小信检索。

### 4.3 认识牧院

面向新生与家长的**「认识牧院」模块**，对应产品定位三大能力之「认识牧院」（见 §1.1）。路由前缀 `/intro`。

#### 结构（三个 Tab）

| Tab | 路径 | 说明 |
|-----|------|------|
| 牧院大百科 | `/intro/wiki` | 学校级介绍（默认页；游客登录后首选） |
| 学院概况 | `/intro/colleges` | 学院卡片列表 → 点击进入学院详情 |
| 社团招新 | `/intro/clubs` | 全校社团列表（与学院详情内嵌社团互补） |

#### 牧院大百科

- 顶部大图 Hero + 校徽 + 校训，亮点数据条叠在 Hero 下方
- 三校区卡片（地址来自学校章程；移动端横向滑动）
- 正文板块由公告 category **`intro_campus_wiki`** 驱动；正文含 `<img>` 时自动图文并排
- 无公告时使用内置占位（数据参考学校官网公开信息）

#### 学院详情（如 `/intro/sie`）

- Hero 封面 + 亮点数据 + 页内锚点（简介 / 师资 / 社团）
- 简介板块：公告 **`intro_sie_overview`**（每条 = 一个区块）
- 师资：`intro_sie_faculty`（标题=姓名，正文=简介，可插图）
- **本院社团**：内嵌展示，不跳转全局社团 Tab；筛选规则在 `frontend/src/constants/intro.ts` 的 `clubFilter`（`clubIds` 优先，暂可用 `categories`）

#### 维护说明

管理员在「校园公告」按上述 category 发布即可更新，无需改代码。学院与社团隶属关系在 `intro.ts` 配置 `clubIds` 后生效。

### 4.4 社团系统

#### 浏览侧

学生进入社团页面后，个人信息卡片自动收起。社团以双列卡片网格展示，每张卡片包含：团徽（可上传自定义图片，无则取首字 + 渐变背景）、社团名称、招新状态标签（招新中/已结束）、一句话简介、指导老师、成员人数。支持按分类筛选（信工团学会 / 校级组织 / 兴趣社团）和关键词搜索（搜名称、简介、指导老师）。点击卡片进入详情页。

#### 详情页

顶部 Hero 区展示团徽 + 名称 + 简介 + 背景大图 + 信息栏（成立年份、成员人数、指导老师、招新时间）。招新状态标签支持管理员点击切换。Tab 切换基本信息/风采展示两个视图。基本信息包含：社团介绍（支持 HTML 富文本）、招新信息（招新对象、招新时间、招新人数、招新要求）、社团荣誉。风采展示为照片网格，点击任意照片弹全屏灯箱，支持左右箭头和键盘切换。移动端自动切换为静态图册模式（避免 3D WebGL 卡顿）。

#### 管理侧

创建/编辑社团支持上传团徽和 Hero 背景图。表单覆盖基本信息（名称、分类、状态、招新时间段、简介 300 字限制）、详细信息（成立年份、成员人数、指导老师、招新对象、招新人数、招新要求、详细介绍 HTML）、风采展示（最多 10 张照片）、联系方式（QQ 群号、QQ 群二维码）。社团管理员只能编辑自己名下的社团，系统管理员可管理全部。

### 4.5 问牧墙（新生互助）

学生发帖提问、学长学姐回答；小信答不上时可引导至问牧墙。支持点赞、采纳、楼主/管理员删帖。

### 4.6 校园导览

桌面端加载 3D 校园模型（Three.js 嵌入页），移动端展示实景图册（田径场、篮球场、网球场、排球场）。

## 五、小信智能助手

### 5.1 多级应答链路

用户输入 → **空消息检查**（返回"请输入问题"）→ **问候识别**（12 种问候语匹配，返回欢迎语）→ **领域检测**（75 个校园关键词词表，不相关则拒答，前端走本地兜底）→ **隐私闸**（打探他人信息拦截 / 未登录拦截 / 管理员问个人信息提示）→ **FAQ 关键词匹配**（文本标准化 + 多维度打分，≥ 阈值直接返回）→ **RAG 向量检索**（Ollama 嵌入 + 余弦相似度 + Ollama 生成回答）→ **前端兜底**（FAQ + 公告 + 社团本地关键词匹配）

### 5.2 FAQ 匹配

从数据库加载所有 FAQ 条目（进程内缓存），对用户输入做文本标准化（去标点、去空格、大小写归一），然后做多维度打分：子串正向匹配（14 分）、反向包含（10 分）、滑动窗口匹配（5 分）、关键词命中（3-5 分）、联想短语匹配（7 分）、字符重叠度（≤4 分）。最高分 ≥ 阈值（默认 8.0）则直接返回。这是小信最快路径，快递在哪、熄灯时间等常见问题均在此命中。

### 5.3 RAG 向量检索

FAQ 未命中时，调用 Ollama embedding API 转为向量，在 `documents` 表中做余弦相似度检索（Python 侧计算），取 top-3。最高分 > 阈值（0.6）则将文档内容拼入 prompt，交给 DeepSeek 或 Ollama 生成回答；≤ 阈值则告知"数据库暂无相关信息"。

### 5.4 前端兜底

后端全部不可用时，前端 `useXinChat` 从 `/api/faq`、`/api/announcements`、`/api/clubs` 拉取全量数据到本地，做纯前端关键词匹配。支持社团搜索（"有没有街舞社"→ 匹配社团名称 + 返回简介和跳转链接）。

### 5.5 语音合成

小信回复气泡旁边有喇叭图标，点击后调用 `/api/tts`，后端用 Edge-TTS（微软免费语音合成，音色 zh-CN-XiaoyiNeural）生成 MP3 流返回，浏览器自动播放。可在聊天面板头部开关全局语音播报。

### 5.6 Agent 降级日志

小信未回答上来的问题自动写入 `logs/agent_fallback.log`（10MB 轮转，保留 3 个备份），供运维补充知识库。

## 六、工程特性

### 6.1 安全
- JWT 鉴权 + 启动时弱密钥检测
- 身份证号 SHA-256 加盐哈希
- 登录限流（30 次/分/ IP）+ 错误锁定（5 次/5 分钟）
- HTTP 安全头 + 日志 PII 脱敏
- 上传文件大小限制（5MB）

### 6.2 性能
- FAQ 进程内缓存，增删自动失效
- 前端预加载 FAQ、公告、社团数据（`usePreload`）
- 移动端关闭 GPU 动画（脉冲光晕、Lottie）
- FAQ 客户端分页（10 条/页）

### 6.3 数据规模
- 业务表：students（仅登录校验）、faq、announcements、clubs、documents、forum 等
- 学生名单：`scripts/init_db.py` 种子数据或运维直接写库，**无 Web/API 名册管理**

### 6.4 部署
- Docker Compose 一键启动（前端 + 后端 + PostgreSQL pgvector）
- 环境变量驱动配置（`.env`）
- 数据库端口映射：宿主机 5434 → 容器 5432
- 支持局域网 CORS 正则放行
